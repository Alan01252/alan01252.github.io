Agricamera.co.uk

Agricamera's entire software environment was built with DevOps in mind from the start. Before any new software was written careful thought was taken to make sure the development environments were immediately reproducible by issuing just a couple of commands using Vagrant and Ansible.

This forethought meant deploying to production was just an extension of the development process. Ansible scripts were created which allowed every aspect of Agricameras architecture to be deployed and re-deployed instantly. This included provisioning of highly available node, web and database servers. As well as the management of 100's of remote devices deployed up and down the country.

Tools

Ansible

After experimenting with a number of DevOps tools over the years Ansible quickly stood out as my favourite. The syntax is easy to reason about, the way it works is intuitive, and it's quick to get up and running for any type of new project.
